# Conventions du monorepo

## Nommage

- Dossiers d'apps en `kebab-case` (ex: `gestion-clinique`)
- Branches: `feature/...`, `fix/...`, `chore/...`

## Organisation

- `apps/<app>/backend`
- `apps/<app>/frontend`
- `apps/<app>/README.md`

## Git

- Message de commit clair: `feat: ...`, `fix: ...`, `chore: ...`
- Pull régulièrement depuis `main`
